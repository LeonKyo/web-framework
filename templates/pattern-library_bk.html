Pattern HTML Structure

All main components will use the following HTML container structure. Components within components will not need this container HTML as they should all be within one set of this container.

Usage:
<div class="page-wrapper">

Full width content are placed here, such as carousel, hero banners, full-width background colour or image, border lines, etc.
Fixed content width are placed within <section> tags, using existing CSS classes to control their width.

<div class="page-container">

The "page-container" class, comes with a default of 20px padding for the sides.

<section>
 <div class="page-container">

Place the content within a <section> tag, to position the content within a fixed centralized width.
You can edit the $max-container-width variable found in the _config_global.scss file.

Markup:
  1. <div class="page-wrapper">
  2.  <div class="page-container"></div>
  3.  <section>
  4.   <div class="page-container"></div>
  5.  </section>
  6. </div>

Breakpoints (For Responsive)

The following table shows an overview of available breakpoints and it's respective devices.

Size Breakpoints Device
Mini up to 479px Phones portrait
Small 480px to 767px Phones landscape
Medium 768px to 959px Tablets portrait
Large 960px to 1199px Desktops & tablets landscape
Xlarge 1200px and larger Large Desktops

Usually we build based on 3 breakpoints only, having Mini, Small and Large (which is 768px and larger).

Category Descriptions
